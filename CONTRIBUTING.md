# Contributing

Thanks for your interest in contributing.

## Quick Start

1. Fork the repo and create a feature branch.
2. Make your changes with clear commit messages.
3. Run tests and linters you touched or impacted.
4. Open a pull request describing what changed and why.

## Development

- Use `make` targets where available for builds and tests.
- Keep YAML and Go formatting consistent with the existing style.

## Reporting Bugs

Please include:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Relevant logs or manifests
